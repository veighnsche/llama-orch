# rbee UI Component Library

Welcome to the **rbee UI** component library! This is the shared design system for all rbee applications, ensuring consistent user experience across the platform.

## Design Philosophy

The rbee design system is built around:

- **Honeycomb Yellow/Gold** (#f59e0b) - Our primary brand color inspired by bees
- **Professional yet approachable** - Enterprise-grade but not corporate
- **Data-focused** - Clear information hierarchy
- **Privacy-first** - Security and trust communicated through design

## Component Categories

### Atoms
Basic building blocks like buttons, inputs, and labels.

### Molecules
Simple combinations of atoms like form fields with labels.

### Organisms
Complex UI components like navigation bars and cards.

### Patterns
Complete page layouts and templates.

## Getting Started

Browse the components in the sidebar to see examples, documentation, and interactive demos.

Each component includes:
- **Props documentation** - All available props and their types
- **Interactive controls** - Test different prop combinations
- **Code examples** - Copy-paste ready code snippets
- **Accessibility notes** - ARIA labels and keyboard navigation

## Design Tokens

All components use design tokens for consistent theming. Key token categories:

- **Colors** - Primary, secondary, background, foreground
- **Spacing** - Consistent padding and margins
- **Typography** - Font sizes, weights, and line heights
- **Borders** - Radius and widths
- **Shadows** - Elevation levels

## Usage

```tsx
import { Button } from '@rbee/ui/atoms';

function MyComponent() {
  return <Button variant="primary">Click me</Button>;
}
```

## Contributing

When adding new components:

2. Use design tokens for all styling
3. Include comprehensive prop documentation
4. Add interactive Storybook stories
5. Ensure accessibility compliance
